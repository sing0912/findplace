# Plan - 일회성 설계/개선 지침

## 개요

이 폴더는 기능 기획, 설계, 개선에 대한 **일회성 지침**을 관리합니다.
설계가 완료되고 구현이 끝나면 `archive/` 폴더로 이동합니다.

---

## 파일 명명 규칙

```
{YYYY-MM-DD}-{topic}.md
```

예시:
- `2026-01-24-initial-design.md`
- `2026-02-01-supplier-settlement-improvement.md`
- `2026-02-15-inventory-warehouse-extension.md`

---

## 워크플로우

```
1. 기능 기획/개선 요청
       ↓
2. docs/plan/{date}-{topic}.md 작성
       ↓
3. 설계 검토 & 승인
       ↓
4. docs/develop/{domain}/ 영구 지침 업데이트
       ↓
5. 구현
       ↓
6. 지침 vs 구현 검증
       ↓
7. 플랜 파일 → archive/ 이동
```

---

## 플랜 문서 템플릿

```markdown
# {제목}

## 배경
- 왜 이 기능/개선이 필요한가?

## 목표
- 무엇을 달성하려고 하는가?

## 범위
- 포함: 이번에 구현할 것
- 제외: 이번에 구현하지 않을 것

## 설계

### 도메인 모델
- 엔티티, 관계

### API 스펙
- 엔드포인트, 요청/응답

### 비즈니스 로직
- 핵심 로직 설명

### 예외 처리
- 에러 케이스

## 영향 범위
- 수정이 필요한 기존 코드/지침

## 테스트 계획
- 테스트 케이스

## 일정
- 마일스톤 (선택)
```
