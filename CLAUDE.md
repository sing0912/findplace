# Claude Code 프로젝트 지침

## 필수 참조 문서

**모든 작업 시작 전 반드시 `AGENTS.md`를 먼저 읽고 참조하세요.**

```
AGENTS.md  ← 프로젝트 메인 지침 (필수 참조)
```

## 참조 우선순위

1. **AGENTS.md** - 프로젝트 전체 규칙 및 필수 지침
2. **docs/develop/{domain}/README.md** - 작업 도메인별 영구 지침
3. **docs/plan/** - 진행 중인 설계/개선 지침

## 작업 워크플로우

1. `AGENTS.md` 읽기 (필수 규칙 확인)
2. 관련 도메인 지침 확인 (`docs/develop/{domain}/`)
3. 진행 중인 플랜 확인 (`docs/plan/`)
4. 영구 지침 업데이트 → 구현 → 검증 순서 준수

## 핵심 원칙

- 영구 지침 없이 구현 금지
- 구현 후 지침과 100% 일치 검증 필수
- "나중에 문서화" 방식 금지

## Git 커밋 메시지

- **커밋 메시지는 반드시 한글로 작성**
- 형식: `[타입] 제목`
- 타입 예시: 기능, 수정, 리팩토링, 문서, 테스트, 설정
- 예시:
  - `[기능] 사용자 로그인 기능 추가`
  - `[수정] 예약 날짜 유효성 검사 오류 해결`
  - `[리팩토링] 결제 서비스 코드 구조 개선`
  - `[문서] API 명세서 업데이트`
